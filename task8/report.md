## Найденные уязвимости

### 🔴 Критические

#### 1. Отсутствие аутентификации и авторизации
**Файл:** `main.py`, все эндпоинты  
**Описание:** Любой пользователь может создать, изменить или удалить анкету репетитора без какой-либо проверки личности. Нет токенов, нет API-ключей, нет ролей.  
**Риск:** Полная компрометация данных — злоумышленник может удалить все анкеты или создать фиктивные.  
**Исправление:** Добавить JWT-аутентификацию через `python-jose` + `passlib`, использовать `Depends(verify_token)` на защищённых эндпоинтах.

---

### 🟠 Высокие

#### 2. Отсутствие валидации email
**Файл:** `main.py`, класс `TutorBase`, поле `email: str`  
**Описание:** Поле принимает любую строку — `"not_an_email"`, `"123"`, `""` — без проверки формата.  
**Риск:** Битые данные в БД, невозможность отправки уведомлений, обход бизнес-логики.  
**Исправление:** Заменить `str` на `EmailStr` из `pydantic` (требует `pydantic[email]`).

```python
# Было:
email: str = Field(..., description="Email репетитора")

# Стало:
from pydantic import EmailStr
email: EmailStr = Field(..., description="Email репетитора")
```

#### 3. Отсутствие защиты от XSS (Cross-Site Scripting)
**Файл:** `main.py`, текстовые поля `full_name`, `subject`, `education`, `bio`  
**Описание:** В текстовые поля можно передать `<script>alert('xss')</script>` — если данные когда-либо попадут в HTML-шаблон, выполнится вредоносный скрипт.  
**Риск:** Кража сессий пользователей, фишинг, дефейс интерфейса.  
**Исправление:** Валидатор, запрещающий HTML-теги в текстовых полях.

```python
@field_validator("full_name", "subject", "education", "bio", mode="before")
@classmethod
def strip_dangerous_chars(cls, v):
    if v and re.search(r"<[^>]+>", v):
        raise ValueError("Поле не должно содержать HTML-теги")
    return v
```

### 🟡 Средние

#### 4. Отсутствие Rate Limiting
**Файл:** `main.py`, все эндпоинты  
**Описание:** Нет ограничения на количество запросов с одного IP. Эндпоинт `POST /tutors` можно вызывать бесконечно.  
**Риск:** DoS-атака, флуд базы данных фиктивными анкетами, брутфорс.  
**Исправление:** Реализовать rate limiting через `slowapi` (в продакшене) или простой in-memory счётчик.

#### 5. Отсутствие CORS-политики
**Файл:** `main.py` — middleware не настроено  
**Описание:** API принимает запросы с любого домена. Браузер не блокирует кросс-доменные запросы к API.  
**Риск:** CSRF-атаки, нежелательное использование API сторонними сайтами.  
**Исправление:**
```python
from fastapi.middleware.cors import CORSMiddleware

app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://tutor-platform.example.com"],
    allow_methods=["GET", "POST", "PUT", "DELETE"],
    allow_headers=["*"],
)
```

#### 6. Отсутствие верхней границы для `hourly_rate`
**Файл:** `main.py`, `hourly_rate: float = Field(..., ge=0)`  
**Описание:** Поле ограничено снизу (≥ 0), но не ограничено сверху. Можно передать `hourly_rate: 9999999999`.  
**Риск:** Мусорные данные в БД, ошибки в расчётах выплат.  
**Исправление:** `Field(..., ge=0, le=100_000)`

---

### 🟢 Низкие

#### 7. Отсутствие логирования
**Файл:** `main.py`, все эндпоинты  
**Описание:** Нет ни одной записи в лог при создании, изменении или удалении данных.  
**Риск:** Невозможно обнаружить атаку или восстановить историю изменений.  
**Исправление:** Добавить `logging` с записью IP, действия и ID ресурса на каждый запрос.

#### 8. In-memory хранилище без защиты от race condition
**Файл:** `main.py`, `tutors_db: dict`, `_next_id: int`  
**Описание:** При конкурентных запросах возможна гонка при чтении/записи `_next_id`. Данные теряются при перезапуске.  
**Риск:** Дублирование ID, потеря данных.  
**Исправление:** Использовать реальную СУБД (PostgreSQL + SQLAlchemy) с автоинкрементом.
